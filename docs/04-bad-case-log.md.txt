# Bad Case 记录

## Product Case 001｜删除源数据后历史评测记录失去关联

### 现象
删除测试用例后，Bad Case 页面仍保留对应评测记录，但标题显示为“已删除的用例”，Prompt 显示为“已删除版本”。

### 原因
EvalResult 只保存 testCaseId 和 promptVersionId，页面展示时依赖当前测试用例和 Prompt 版本表。源数据删除后，历史记录无法还原当时内容。

### 产品判断
评测记录属于历史证据，不应因为测试用例或 Prompt 被删除而失真。因此历史评测记录应该保存当时的测试用例和 Prompt 快照。

### 解决方案
- 保存评测记录时，同时保存测试用例标题、输入、期望答案、评分标准、Prompt 名称和 Prompt 内容快照
- Bad Case 和评测报告优先读取快照字段
- 增加 runMode 字段区分 mock 和 real 数据
- 增加 Bad Case 删除按钮，允许用户清理误保存或无效记录

### 价值
保证评测记录可追溯，避免历史报告依赖可变源数据，为后续真实模型评测和 Prompt 版本对比打基础。
## Product Case 001｜已修复：删除源数据后历史评测记录失去关联

### 修复结果
通过在 EvalResult 中保存测试用例和 Prompt 版本快照，解决了删除源数据后历史记录无法正常展示的问题。

### 验收结论
- 删除测试用例后，历史评测记录仍可展示原始测试标题与输入内容
- 删除 Prompt 后，历史评测记录仍可展示原始 Prompt 名称
- Bad Case 支持手动删除
- 删除后报告统计同步更新

### 面试可讲点
这个问题体现了评测系统里的“历史记录不可变性”。评测结果不是普通临时数据，而是一次测试的历史证据，因此需要保存当时的输入、Prompt 和评分结果快照，保证后续报告可追溯。
