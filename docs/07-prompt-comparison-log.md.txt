# V1.2｜Prompt 对比分析

## 目标
基于历史评测记录，对比不同 Prompt 版本在同一测试用例、同一模型、同一 Judge 条件下的表现差异。

## 完成内容
- 支持选择测试用例、模型、Judge、Prompt A、Prompt B
- 统计平均总分、五维评分、Bad Case 数、Bad Case 率、平均耗时、平均 Token
- 展示 Prompt A / Prompt B 的代表记录
- 自动生成规则型结论，不调用 AI
- 结论综合考虑效果、成本和响应速度

## 产品价值
评测记录本身只能说明单次结果，Prompt 对比分析可以帮助判断 Prompt 修改是否真正带来效果提升，并避免“分数略高但成本大幅增加”的错误决策。

## 示例观察
在“童年经历事实提取”任务中，Prompt V1 与 Prompt V3 分数接近，但 Prompt V3 的平均 Token 和耗时明显更高，因此综合效果、成本和响应速度，Prompt V1 更适合作为当前默认版本。
## V1.3｜真实串行批量评测

### 完成内容
- 支持选择多个测试用例
- 使用同一个 Prompt、模型、temperature 和 AI Judge 设置进行批量评测
- 按顺序逐条执行真实模型调用和 AI Judge 评分
- 单条失败不会中断整个批次
- 每条结果自动保存为 EvalResult
- 每条记录保存 batchId、batchName、batchIndex、batchTotal
- 评测详情页支持查看批量评测信息

### 产品判断
批量评测是 AI Eval 工作流的核心能力。相比单条调试，批量评测可以让用户一次性验证 Prompt 在多个测试用例上的稳定性，并通过评测报告和 Prompt 对比分析观察整体效果。
