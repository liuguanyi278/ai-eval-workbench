# 功能迭代记录

## V0 Mock 原型｜AI 产品评测工作台

### 完成内容
- 搭建 AI 产品评测工作台基础页面
- 完成项目概览、测试用例、Prompt 版本、Prompt 测试台、模型能力矩阵、Bad Case、评测报告模块
- 使用 Mock 逻辑模拟模型输出和 AI 评分
- 使用 localStorage 保存本地数据

### 验收结果
- 新增测试用例后，刷新页面数据不丢失
- 新增 Prompt 版本后，Prompt 测试台下拉框可选
- Prompt 测试台可以生成 Mock 模型输出
- AI 模拟评测可以生成评分、Bad Case 类型和优化建议
- 保存评测记录后，评测报告可以同步更新

### 发现问题
- 删除测试用例后，Bad Case 页面仍保留历史评测记录，但显示为“已删除的用例”
- 删除 Prompt 版本后，历史记录显示为“已删除版本”
- Mock 模型输出和 Mock AI 评分容易让用户误以为是真实模型调用
- Bad Case 页面没有手动删除按钮，无法清理无效或误保存的记录

### 下一步
修复评测记录快照机制、Mock/Real 标识、历史记录展示逻辑，并增加 Bad Case 删除能力。
## V0.1｜评测记录快照与 Bad Case 管理修复

### 完成内容
- 修复删除测试用例后 Bad Case 显示“已删除的用例”的问题
- 修复删除 Prompt 版本后历史评测记录显示“已删除版本”的问题
- 为评测记录增加测试用例与 Prompt 版本快照
- 增加 Mock / Real 数据标识
- 增加 Bad Case 删除按钮
- 删除 Bad Case 后，评测报告统计可同步更新
- 刷新页面后数据仍可保留

### 验收结果
- 保存 Mock 评测记录后，删除关联测试用例，Bad Case 仍显示原始标题
- 删除关联 Prompt 后，Bad Case 仍显示原始 Prompt 名称
- 点击 Bad Case 删除按钮后，对应评测记录被删除
- 评测报告统计同步变化
- 页面刷新后数据不丢失

### 产品判断
评测记录属于历史证据，不能依赖当前测试用例和 Prompt 版本是否仍然存在。因此在保存评测记录时写入内容快照，使历史记录和源数据解耦。

### 下一步
进入 V1 阶段：接入真实模型 API，让 Prompt 测试台从 Mock 输出升级为真实模型输出。

## 首页定位调整｜区分产品定位与示例项目

### 问题
首页 hero 区域直接展示“老人自传生成工作流评测”，容易让用户误以为该产品只服务老人自传项目。

### 调整
将首页主标题改为“AI 产品评测工作台”，突出产品的通用定位；将“老人自传生成工作流评测”降级为当前示例项目。

### 产品判断
评测工作台应作为通用 AI 产品评测框架存在，老人自传项目只是第一个验证案例，用来展示测试用例、Prompt 版本、Bad Case 和报告闭环。
